Try 2!

可以不需要再開一個自己的repo作業，因為已經在對方的分支上有自己的進度了

想要自己操作給自己看不給別人看的話：

git remote add 取新的貼紙名

因為剛剛clone別人所以你的origin被用走了！ 之後再改個push指令確保我們都是在自己的repo上作業 

git push -u myrepo dev

萬萬記得在推以前都要先fetch 一下對方 確保是最新進度

不merge改rebase

還是要先記得先fetch遠端的最新資料抓取但不合併 這樣做可以同步到本地但不會改動？！ 你在什麼分支，就關心同名的遠端分支

把git rebase 想成我不要自己是岔出去的線， 我要把我岔出去的東東接到原本你後面才漂亮

'你正在操作的分支' git rebase 'origin/dev' 有夠卡這裡為啥是對應這個分支啦==???

注意！！！之後都要在對方的分支上工作開發功能
