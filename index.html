'你正在操作的分支' git rebase 'origin/dev' 有夠卡這裡為啥是對應這個分支啦==???

注意！！！之後都要在對方的分支上工作開發功能

所以每次開始寫前都要對齊！

我剛剛去處理main沒有對齊遠端main的問題，使用了

git reset --hard origin/main

怪死了

這樣子才是成功讓main merge dev 後的狀態

Try 7!

太棒了可以去任何想去的地方

git checkout -B '分支名' 想去的commitID(4碼) git switch -c '分支名' 想去的commitID(4碼)

你的本端到了，可是你也需要修正遠端的位置，因此需要[覆蓋歷史紀錄] git push origin demo --force 這樣就能把目前本地端的狀況覆蓋遠端了
